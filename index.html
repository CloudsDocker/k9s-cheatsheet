01

Launch & flags

starting k9s the right way
  • k9s
    Launch k9s using your current kubeconfig context.
  • k9s -n NS
    Start in a specific namespace.
  • k9s -c CTX
    Start with a specific context from kubeconfig.
  • k9s --context NAME
    Longer form of -c.
  • k9s --readonly
    Launch in read-only mode. Disables edits, deletes, exec. prod-safe
  • k9s --headless
    Hide the top-status header for more screen real estate.
  • k9s --logoless
    Hide the k9s logo.
  • k9s --crumbsless
    Hide the breadcrumb trail at the bottom.
  • k9s -A
    Launch viewing all namespaces.
  • k9s -r 3
    Set refresh rate (seconds). Default is 2.
  • k9s --kubeconfig PATH
    Use a specific kubeconfig file.
  • k9s info
    Print version and paths (config, logs, plugins, skins).
  • k9s version
    Print version only.
02

Universal navigation

these work almost everywhere in the app
  • :
    Enter command mode. Type a resource alias (pod, svc, ctx…) then Enter.
  • /
    Enter filter mode for the current view.
  • ?
    Show the help panel with keys available in the current view.
  • Ctrl+a
    Show all aliases — every resource shortcut k9s knows.
  • Esc
    Close a menu / cancel filter / go back one view.
  • Enter
    Drill into the selected row (e.g. pod → containers).
  • Tab
    Move between panes in multi-pane views.
  • ↑/↓ j/k
    Move selection up and down (vim-style works too).
  • PgUp/PgDn
    Page through long lists.
  • g·G
    Jump to top / bottom of the list.
  • :q · :quit · Ctrl+c
    Quit k9s.
03

Jump to resources

type : then the alias, then Enter
  • :pod · :po
    Pods.
  • :deploy · :dp
    Deployments.
  • :svc
    Services.
  • :ing
    Ingresses.
  • :ns
    Namespaces.
  • :node · :no
    Nodes.
  • :cm
    ConfigMaps.
  • :sec
    Secrets (values are base64 — press x to decode).
  • :pvc · :pv
    Persistent volume claims / volumes.
  • :sts
    StatefulSets.
  • :ds
    DaemonSets.
  • :rs
    ReplicaSets.
  • :job · :cj
    Jobs / CronJobs.
  • :sa
    ServiceAccounts.
  • :role · :rb · :crb
    Roles, RoleBindings, ClusterRoleBindings.
  • :events · :ev
    Events across the cluster — watch here when things break. debug
  • :ctx
    Switch kubeconfig context without leaving k9s.
  • :helm
    Helm releases. Enter to see revision history.
  • :pf
    Active port-forwards managed by this k9s session.
  • :screendump · :sd
    Dump current cluster state (YAMLs) to disk for offline review.
  • :alias
    Full aliases list — same as Ctrl+a.
tip · fuzzy jumping

You can jump directly to a single resource: :pod mypod-abc123 opens that pod view. You can also combine namespace + resource: :pod production filters pods to the production namespace.

04

Resource actions

with a row highlighted, press…
  • d
    Describe — full kubectl-describe output.
  • y
    View the resource YAML as the API returns it.
  • e
    Edit — opens YAML in $EDITOR; save to apply.
  • l
    Logs for a pod (or pod-backed resource).
  • p
    Previous container logs — the last crash.
  • s
    Shell into a pod (picks container if multiple).
  • a
    Attach to a running container's stdin/stdout.
  • f
    Show existing port-forwards for this pod / service.
  • Shift+f
    Create a port-forward (prompts for ports).
  • r
    On a deployment: restart rollout (kubectl rollout restart).
  • x
    On secrets: decode a value. Elsewhere: resource-specific action.
  • t
    On deployments: scale (transition). Prompts for replica count.
  • Space
    Mark the current row. Multi-select across rows.
  • Ctrl+Space
    Clear all marks.
  • Ctrl+d
    Delete with confirmation. Respects marks — can bulk-delete.
  • Ctrl+k
    Kill — delete without confirm, with --grace-period=0 --force. danger
05

Logs viewer

keys inside the log pane
  • 0
    Tail all lines (since pod start, follow indefinitely).
  • 1 … 5
    Tail the last 1, 10, 100, 1000, 5000 lines respectively.
  • f
    Toggle full screen logs.
  • w
    Toggle line wrap.
  • t
    Toggle timestamps.
  • s
    Save current buffer to disk (path printed at top).
  • c
    Copy current buffer to clipboard.
  • m
    Mark current position — useful before triggering an event.
  • n · Shift+n
    Jump to next / previous mark.
  • Shift+c
    Clear the log buffer (display only; underlying logs remain).
  • Ctrl+s
    Toggle autoscroll.
  • /
    Filter log lines (regex supported).
06

Filter & sort

narrow the list, then order it
  • /text
    Filter rows that contain "text".
  • /!text
    Inverse filter — exclude rows matching "text".
  • /-l app=web
    Filter by label selector.
  • /-f status.phase=Running
    Filter by field selector.
  • / regex
    Regex works too (e.g. /api-.*-prod).
  • Shift+n
    Sort by name.
  • Shift+a
    Sort by age.
  • Shift+s
    Sort by status.
  • Shift+r
    Sort by ready state.
  • Shift+c
    Sort by CPU usage (requires metrics-server).
  • Shift+m
    Sort by memory usage.
  • Shift+i
    Sort by IP.
  • Shift+o
    Sort pods by node — handy for noisy-neighbour hunting.
  • 0
    In a namespaced view: show all namespaces.
  • 1 … 9
    Jump to your favourite namespace (configured via :ns then Ctrl+s).
07

Namespaces & contexts

switching realms without leaving the app
  • :ns
    Open namespace list. Enter to switch to one.
  • Ctrl+s
    In namespace view: favourite the selected namespace. Accessible by digit keys thereafter.
  • u
    In namespace view: unfavourite.
  • :ctx
    List kubeconfig contexts. Enter to switch — faster than kubectl config use-context.
  • 0
    View all namespaces (any resource list).
08

Port forwarding

local ports tunnelled to cluster resources
  • Shift+f
    Create a port-forward on the selected pod or service. Prompts for local:container ports.
  • :pf
    List active port-forwards managed by this k9s process.
  • Ctrl+b
    On a forward: open the URL in the default browser.
  • Ctrl+d
    Delete / stop the selected forward.
note · forward lifetime

Port-forwards live only as long as the k9s process. Quit k9s and the tunnels close — same as kubectl port-forward.

09

Cluster tools

higher-order views
  • :pulse
    Cluster health dashboard — resource counts, usage, node status at a glance.
  • :popeye
    Run Popeye — a cluster sanitizer that flags misconfigurations and best-practice violations. must-try
  • :xray RESOURCE
    Visualise resource dependency tree. Example: :xray deployment. Drill into pods, services, configmaps, secrets they depend on.
  • :dir PATH
    Browse and apply YAML manifests from a local directory (dry-run or apply).
  • :benchmark · :be
    Run / inspect benchmarks configured in bench-<context>.yaml.
  • :plugins
    List configured plugins (extra key bindings you've defined).
  • :skins
    List available skins. Useful to visually distinguish prod vs dev clusters.
10

Config & paths

where k9s stores its brain
  • k9s info
    Prints exact paths on your system (works cross-platform). Run this first.
  • config.yaml
    Global settings: refresh rate, default namespace, UI toggles.
  • aliases.yaml
    Custom resource aliases. Add your own: e.g. pp: v1/pods.
  • hotkeys.yaml
    Global key bindings. Map e.g. Shift+x to jump to a favourite resource.
  • plugins.yaml
    Per-view commands that shell out to scripts. E.g. Shift+k on a pod to run a custom diagnostic.
  • skins/*.yaml
    Per-cluster colour themes. Reference by context name in config to auto-apply.
  • ~/.config/k9s
    Linux / XDG default location.
  • ~/Library/Application Support/k9s
    macOS default location.
  • %LOCALAPPDATA%\k9s
    Windows default location.
tip · prod-safe skins

Make production red. Drop a prod.yaml skin with alarming colours in skins/, then reference it by context in config.yaml. You'll never confuse prod and dev again.

11

Everyday workflows

the sequences you'll run daily
Tail logs from a specific pod
Goal: follow a running pod's logs, filtered to errors.
  1. :pod Enter — open the pod list.
  2. /, type part of the pod name, Enter — filter.
  3. l — open logs.
  4. 0 — tail indefinitely from pod start.
  5. / error Enter — filter to error lines.
Exec into a pod
Goal: get a shell in a running container.
  1. :pod Enter.
  2. Highlight the pod row.
  3. s — if multiple containers, select one.
  4. Exit with Ctrl+d or exit.
Port-forward a service to localhost
Goal: hit an in-cluster service from your laptop.
  1. :svc Enter.
  2. Highlight the service row.
  3. Shift+f — prompts for local port.
  4. Accept defaults or type 8080:80. Enter.
  5. :pf to confirm it's live.
Edit a ConfigMap and roll the deployment
Goal: change config and make pods pick it up.
  1. :cm Enter, select your ConfigMap.
  2. e — edit in your editor. Save.
  3. :deploy Enter, select the consuming deployment.
  4. r — restart rollout.
Find what's failing right now
Goal: triage after a deploy or outage.
  1. :ev Enter — watch cluster events.
  2. Shift+a — sort by age (newest first).
  3. / Warning — filter to warnings/errors.
  4. :pulse to correlate with overall cluster health.
Decode a secret value
Goal: read a base64 secret without shell gymnastics.
  1. :sec Enter.
  2. Highlight the secret, Enter to open.
  3. x on the selected key — decoded in-place.
Bulk-delete stuck pods
Goal: remove many pods in one go.
  1. :pod Enter.
  2. / filter to the target pods.
  3. Space on each to mark.
  4. Ctrl+d to delete with confirm — or Ctrl+k to force-kill.
12

Gotchas & tips

the stuff that only shows up after hours of use
  • Ctrl+k vs Ctrl+d
    Ctrl+k is force-kill without confirmation. Ctrl+d asks first. Muscle memory matters. danger
  • --readonly
    Run prod clusters read-only. A single flag prevents a career-ending keystroke. habit
  • 0 in lists
    0 means "all namespaces" in resource lists — and "tail all" in log views. Context-sensitive.
  • Space is sticky
    Marks persist across filter changes. If you're getting unexpected bulk-deletes, Ctrl+Space clears all marks.
  • Metrics missing
    CPU / memory columns require metrics-server installed in the cluster. Otherwise you'll see dashes.
  • Edit uses $EDITOR
    Set it to something you know: export EDITOR=nvim. Default may drop you in vi.
  • :xray with RBAC
    If xray shows empty trees, you probably lack RBAC for referenced resources — not a k9s bug.
  • Screen dumps
    :sd writes YAML per resource to disk — great for "what did the cluster look like 20 minutes ago" post-mortems.
  • Helm rollback
    In :helm, Enter shows revision history, then select a revision and u to roll back.
  • Favourites work globally
    Favourite a namespace once with Ctrl+s, then digit keys 1–9 switch to it from anywhere.
